## Supported IDEs

/vibe works in any IDE that supports the Model Context Protocol (MCP):

| IDE | Status | Config Location |
|-----|--------|-----------------|
| **Claude Code** | Full support | `~/.claude.json` |
| **Cursor** | Full support | `~/.cursor/mcp.json` |
| **VS Code (Copilot)** | Full support | `.vscode/mcp.json` |
| **Windsurf** | Compatible | `~/.codeium/windsurf/mcp_config.json` |
| **Cline** | Compatible | MCP Servers panel |
| **Continue.dev** | Compatible | `.continue/mcpServers/vibe.json` |
| **JetBrains** | Compatible | Settings > Tools > AI Assistant > MCP |

## Auto-detect all editors

The fastest way to set up multiple editors at once:

```bash
npx slashvibe-mcp install
```

This detects which editors you have installed and configures all of them.

## Claude Code

Run in your terminal:


## VS Code (GitHub Copilot)

Add to `.vscode/mcp.json` in your workspace:

```json
{
  "servers": {
    "vibe": {
      "command": "npx",
      "args": ["-y", "slashvibe-mcp"]
    }
  }
}
```

## Cline / Continue.dev / JetBrains

**Cline:** Open MCP Servers > Configure > Edit JSON, add the vibe server config above.

**Continue.dev:** Create `.continue/mcpServers/vibe.json`:

```json
{
  "command": "npx",
  "args": ["-y", "slashvibe-mcp"]
}
```

**JetBrains:** Settings > Tools > AI Assistant > MCP, add server with command `npx` and args `-y slashvibe-mcp`.

## What MCP tools are available?

Once connected, your AI has access to these /vibe tools:

- `vibe_who` — See connected developers
- `vibe_dm` — Send a direct message
- `vibe_inbox` — Check your messages
- `vibe_ship` — Share what you've built
- `vibe_game` — Play multiplayer games
- `vibe_discover` — Find developers to pair with
- `vibe_doctor` — Run diagnostics

See the full [MCP Tools Reference](/building/mcp-tools) for all 39 tools.
